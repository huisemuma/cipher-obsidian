# Cipher Obsidian

在 Obsidian 笔记中安全地存储和查看密码。

密码以密文形式写入笔记文件，只有在你主动查看时才会解密，且解密过程完全在本地内存中完成——主密码不落盘、不同步、不上传。

## 效果

笔记源文件中的内容：

```
Google: `🔐VGhpcyBpcyBhIHRlc3Q`
```

阅读模式下显示为：

```
Google: ●●●●●●●●  📋 👁
```

点击 📋 直接复制密码，点击 👁 查看明文。

## 使用

**加密插入密码：** `Cmd+P`（手机端下滑打开命令面板）→ 搜索 "Lock" → 输入要加密的密码和主密码 → 密文自动插入光标处

**查看/复制密码：** 切换到阅读模式 → 点击 📋 复制到剪贴板，或点击 👁 显示明文

**重新输入主密码：** 查看密码后出现 🔄 按钮，点击可清除缓存的主密码并重新输入

## 加密原理

### 密钥派生

使用 [PBKDF2](https://en.wikipedia.org/wiki/PBKDF2) 从主密码派生加密密钥：

- 哈希算法：SHA-256
- 迭代次数：100,000
- 固定盐值：`cipher-v1`

100,000 次迭代意味着即使攻击者拿到密文，暴力破解每次猜测都需要执行 10 万次哈希运算，极大增加破解成本。

### 加密过程

1. **编码长度**：将明文长度编码为一个字符，拼接在明文前面
2. **随机填充**：用随机字符填充到固定 18 位，使相同明文每次加密结果不同
3. **逐字符加密**：对每个字符执行模运算加密

```
加密：encrypted = ((char - 33 + key) % 94 + 94) % 94 + 33
解密：decrypted = ((char - 33 - key) % 94 + 94) % 94 + 33
```

运算范围覆盖 ASCII 33(`!`) 到 126(`~`) 共 94 个可打印字符。

### 存储格式

密文经过 Base64url 编码后嵌入行内代码：

```
`🔐<24位Base64url字符>`
```

Base64url 只使用字母、数字、`-`、`_`，避免与 Markdown 语法冲突。

## 安全设计

### 主密码不离开内存

主密码存储在插件的 JavaScript 变量中，**不会写入任何文件**。Obsidian 的 `data.json` 只保存超时时长设置，不包含密码。关闭 Obsidian 后内存释放，密码即消失。

### 不修改笔记文件

解密只修改页面渲染（DOM），**不会回写 `.md` 文件**。笔记文件中始终只有密文，无论是 Obsidian Sync、iCloud 还是 Git 同步，传输的都是密文。

### 自动过期

主密码在设定时间后自动清除（默认 5 分钟），需重新输入。也可设为 0 表示不过期（直到关闭 Obsidian）。

### 密文不可逆推明文

没有主密码，密文无法被解密。PBKDF2 的高迭代次数使得暴力破解在计算上不可行。

### 威胁模型

| 场景 | 是否安全 |
|---|---|
| 笔记文件被他人看到 | 安全 — 只能看到密文 |
| 同步服务被入侵 | 安全 — 同步的只有密文 |
| 设备丢失（有锁屏） | 安全 — 内存中无密码残留 |
| 设备被安装恶意软件 | 不安全 — 恶意软件可读取进程内存或键盘输入 |

## 安装

将以下 3 个文件复制到 Obsidian vault 的插件目录：

```
<vault>/.obsidian/plugins/cipher-decode/
├── main.js
├── manifest.json
└── styles.css
```

然后在 Obsidian 设置 → 第三方插件 中启用 "Cipher Decode"。

### 从源码构建

```bash
git clone https://github.com/huisemuma/cipher-obsidian.git
cd cipher-obsidian
npm install
npm run build
```

构建产物为 `main.js`，连同 `manifest.json` 和 `styles.css` 一起复制到插件目录即可。

## 设置

| 选项 | 说明 | 默认值 |
|---|---|---|
| Password timeout | 主密码在内存中保留的时间（分钟），0 = 不过期 | 5 分钟 |

## 兼容性

本插件的加密算法与配套的 [Cipher 网页工具](https://github.com/huisemuma/cipher-obsidian) 完全一致，两者加密的密文可以互相解密。
